# sites-odsports-guide

本仓库用于归档和发布多个独立的 HTML 页面，主要面向体育相关信息的展示与导航用途。仓库不绑定任何特定域名或具体网站，仅提供静态页面内容的存储与版本管理。

## 项目简介

该仓库旨在以结构化方式保存一组 HTML 文件，便于通过 GitHub Pages 或其他静态托管服务进行访问。所有页面均为独立编写，内容侧重体育资讯的聚合与呈现，适合作为信息索引或参考页面。

## 目录说明

```
/
├── index.html          # 主入口页面
├── pages/              # 各独立页面存放目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 静态资源（CSS、JS、图片等）
├── README.md           # 本文件
└── CNAME               # 自定义域名配置（可选）
```

- `pages/`：存放所有独立 HTML 页面，文件名无固定规则，按需命名。
- `assets/`：存放页面引用的样式、脚本、字体或图片等资源文件。
- `CNAME`：如需绑定自定义域名，可在此文件内填写域名（本仓库默认不启用）。

## 页面归档说明

- 每个 HTML 文件均为自包含页面，可独立运行。
- 页面之间无强依赖关系，但可能共享部分公共资源（如统一风格的 CSS）。
- 归档内容会不定期更新，新增或修改页面时尽量保持向后兼容。
- 历史版本可通过 Git 提交记录追溯，不建议直接修改已发布的页面结构。

## 维护说明

- 欢迎提交 Issue 反馈页面显示问题或提出改进建议。
- 如需贡献新的 HTML 页面，请确保文件命名清晰，并在 `pages/` 目录下添加。
- 提交 Pull Request 时请附上简要说明，包括页面用途和内容来源（如有）。
- 不建议在该仓库中存放敏感信息、商业性质内容或受版权保护的材料。
